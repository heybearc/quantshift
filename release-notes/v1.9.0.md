# QuantShift v1.9.0 Release Notes

**Release Date:** March 4, 2026  
**Release Type:** Feature Release - Phase 3 Advanced Risk Management

---

## 🎯 Overview

Version 1.9.0 completes **Phase 3: Advanced Risk Management** with circuit breaker email alerts and a comprehensive risk management dashboard. This release provides real-time risk monitoring, automated email notifications for critical events, and manual circuit breaker controls.

---

## ✨ New Features

### 1. **Circuit Breaker Email Alerts**

Automated email notifications when risk limits are breached:

- **Comprehensive Alert Content:**
  - Circuit breaker type (Daily Loss Limit or Max Drawdown)
  - Current risk metrics vs. limits
  - Account equity and daily P&L
  - Open positions table with unrealized P&L
  - Action-required steps for recovery
  
- **Smart Notification Logic:**
  - Sent once per trigger to prevent spam
  - Resets automatically on new trading day
  - Includes visual indicators (red header, metrics grid)
  - Sent to error recipients for critical visibility

### 2. **Risk Management Dashboard** (`/risk`)

Real-time risk monitoring interface with:

- **Portfolio Heat Gauge:**
  - Current risk exposure as percentage
  - Utilization bar with color-coded warnings
  - Limit tracking and visual indicators
  
- **Daily P&L Tracking:**
  - Real-time profit/loss monitoring
  - Percentage-based limit tracking
  - Color-coded positive/negative display
  
- **Drawdown Monitor:**
  - Peak equity tracking
  - Current drawdown percentage
  - Progress bar showing limit utilization
  
- **Circuit Breaker Status:**
  - Visual status indicator (green/yellow/red)
  - Current status reason display
  - Manual reset button (admin only)
  
- **Auto-Refresh:**
  - Updates every 30 seconds
  - Manual refresh button available
  - Real-time risk metrics from Redis

### 3. **Risk Management API Endpoints**

New API routes for risk data:

- **GET `/api/bot/risk-metrics`**
  - Fetches current risk metrics from Redis
  - Fallback to mock data if Redis unavailable
  - Returns portfolio heat, daily P&L, drawdown, circuit breaker status
  
- **POST `/api/bot/reset-circuit-breaker`**
  - Sends reset signal to bot via Redis
  - Requires manual confirmation
  - Admin-only operation

## 🔒 Risk Controls Summary

### Portfolio Heat
- **Limit:** 10% of account equity
- **Action:** Blocks new trades if exceeded
- **Warning:** Alert at 80% utilization

### Daily Loss Limit
- **Limit:** 5% of starting equity
- **Action:** Triggers circuit breaker, sends email
- **Warning:** Alert at 80% of limit

### Max Drawdown
- **Limit:** 15% from peak equity
- **Action:** Triggers circuit breaker, sends email
- **Warning:** Alert at 80% of limit

### Circuit Breaker Behavior
- Halts all new trade signals
- Existing positions remain open
- Requires manual review and reset
- Email notification sent immediately

## 📝 Configuration

### Email Alerts

Circuit breaker emails are sent using existing EmailService configuration:

```python
# In bot initialization
email_service = EmailService(
    smtp_server=os.getenv('EMAIL_SMTP_SERVER'),
    smtp_port=int(os.getenv('EMAIL_SMTP_PORT', 587)),
    username=os.getenv('EMAIL_USERNAME'),
    password=os.getenv('EMAIL_PASSWORD'),
    from_email=os.getenv('EMAIL_FROM'),
    from_name=os.getenv('EMAIL_FROM_NAME', 'QuantShift')
)

# Pass to RiskManager
risk_manager = RiskManager(
    max_portfolio_heat=0.10,
    daily_loss_limit=0.05,
    max_drawdown_limit=0.15,
    email_service=email_service  # Enable email alerts
)
```

## 🎓 User Guide

### Monitoring Risk

1. Navigate to **Risk Management** in main menu
2. Review current metrics:
   - Portfolio Heat utilization
   - Daily P&L vs. limit
   - Drawdown from peak
3. Monitor circuit breaker status indicator

### Handling Circuit Breaker Triggers

1. **Email Alert Received:**
   - Review risk metrics in email
   - Check open positions table
   - Assess market conditions
   
2. **Dashboard Review:**
   - Navigate to `/risk` page
   - Review detailed metrics
   - Check circuit breaker reason
   
3. **Manual Reset (if safe):**
   - Click "Reset Circuit Breaker" button
   - Confirm action
   - Bot resumes normal operation

### Best Practices

- Monitor risk dashboard daily
- Review email alerts immediately
- Don't reset circuit breakers without thorough review
- Use warning states (80% of limits) as early indicators
- Adjust position sizes if consistently approaching limits
